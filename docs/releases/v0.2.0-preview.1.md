# ZSUI v0.2.0-preview.1

ZSUI v0.2.0-preview.1 is the first public preview of the Rust-first desktop UI
framework. It is a prerelease and does not mark the v0.2 completion gate as
finished.

## Highlights

- One `State / Msg / view / update` application path for Win32, AppKit and GTK4
  host boundaries.
- Feature-gated component catalog with 48 families: 45 first-pass runtime
  surfaces and 3 contract-only families (`canvas`, `flyout`, `image`).
- Corrected intrinsic sizing for buttons, check boxes and radio buttons so long
  labels are not constrained to the platform minimum width.
- Platform profiles for button, text input, check box, toggle, slider, radio,
  progress, selection and time-picker geometry.
- Compact WinUI control rhythm on Windows, including 32-DIP controls, 4-DIP
  control radius, text-input focus indicator, pointer hover/pressed states and
  content-aware InfoBar height.
- Complete opt-in component Gallery with input, collection, navigation,
  feedback/overlay, layout and catalog pages.
- Buffered no-flicker Win32 drawing, DPI-aware layout, typed focus/input events,
  IME/text editing, menus, clipboard and file-dialog service boundaries.
- No embedded browser or WebView dependency.

## Feature selection

The default feature set remains `window`, `button` and `label`. Individual
components remain independently selectable. The release Gallery uses the
explicit `component-gallery-demo` profile because it intentionally exercises
the complete component set.

```powershell
cargo run --release --example component_gallery --no-default-features --features component-gallery-demo
```

## Evidence in the release archive

- Windows x64 component Gallery executable.
- Five native Win32 screenshots and their machine-readable smoke reports.
- Component catalog JSON and SHA-256 checksums.

AppKit and GTK4 sources remain part of the shared framework boundary. Their
real target screenshots and interaction evidence are still required before the
v0.2 completion gate can be marked complete.
