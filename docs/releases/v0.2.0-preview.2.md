# ZSUI v0.2.0-preview.2

This prerelease corrects the Windows executable type used by the component
Gallery and establishes the same rule for ZSUI's user-facing GUI examples.

## Windows application behavior

- Release-mode GUI examples now use the Windows GUI PE subsystem and do not
  create a console window when launched from Explorer.
- Debug builds keep the console available for diagnostics.
- Command-line smoke and report drivers remain explicit console tools.
- `scripts/check-windows-gui-subsystem.ps1` checks GUI example sources and can
  inspect packaged PE executables for subsystem 2 before publication.
- CI runs the source boundary check on Windows.

The PE subsystem is selected by the final executable crate rather than by a
Rust library dependency. Application entry points therefore use:

```rust
#![cfg_attr(all(windows, not(debug_assertions)), windows_subsystem = "windows")]
```

## Component Gallery

The Windows x64 archive contains the complete optional component Gallery, its
five screenshot pages, component catalog and smoke reports. The Gallery remains
behind `component-gallery-demo`; default ZSUI builds remain limited to `window`,
`button` and `label`.

- Windows TabView headers now place an optional semantic icon and label in one
  WinUI-style row, use the 32-DIP item height and 8-DIP strip inset, and no
  longer draw a generic border around the page content.
- Gallery navigation rows now use semantic icons, left-aligned labels and the
  WinUI-style selection indicator in a 320-DIP expanded pane instead of
  pretending to be ordinary outlined Buttons.
- Compact navigation content no longer receives an equal flexible share of the
  page, and intrinsic Button sizing includes a renderer guard for long labels.
- Breadcrumb segments include the Windows text-measurement guard instead of
  prematurely ellipsizing labels in a wide bar.
- The component catalog now names the remaining DatePicker, TimePicker,
  ColorPicker and adaptive NavigationView parity gaps explicitly.

WebView remains outside the ZSUI product boundary.

## Platform evidence

- Windows: release executable PE header and component Gallery smoke are
  verified locally.
- macOS/Linux: shared source continues to compile for the desktop targets;
  target-machine visual and interaction proof remains pending.
