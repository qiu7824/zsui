# ZSUI v0.2.0-preview.5

## 中文

本预览版增加无闪屏图片预览、同步重排后的分页列表锚点恢复，以及隐藏窗口资源释放策略，并首次提供 crates.io 源码包。

- `image-preview` 使用应用持有的状态、单工作线程、请求合并和 generation 校验解码 PNG；新帧完成前继续显示上一完整帧。
- Win32 图片绘制继续进入同一缓冲目标，与窗口其余内容一次提交；AppKit 与 GTK4 共用相同的保留帧协议。
- 分页虚拟列表在外部同步修订后按稳定条目键和子行像素偏移恢复视口，失效旧索引页并拒绝过期结果。
- `NativeWindowResourcePolicy::ReleaseViewWhenHidden` 可在窗口隐藏或最小化时释放 View 树、绘制/命中计划、文本成形缓存和临时输入状态，重新显示时从保留的应用状态重建。
- Win32、AppKit 和 GTK4 宿主统一接入隐藏/显示生命周期；默认策略仍保留 View，保持向后兼容。
- crates.io 包保留构建所需源码、资源、示例和文档，并排除视频制作素材。

默认 feature 仍为 `window`、`button` 和 `label`。完整组件、图片预览和高级桌面能力继续按 Cargo feature 显式启用。

## English

This preview adds no-flicker image preview, stable viewport recovery after paged-list synchronization, an opt-in hidden-window resource policy, and the first crates.io source package.

- `image-preview` decodes PNG frames on one owned worker with request coalescing and generation checks while retaining the previous complete frame until replacement is ready.
- Win32 presents images through the existing buffered paint target; AppKit and GTK4 consume the same retained-frame protocol.
- Paged virtual lists recover synchronized viewports by stable item key and sub-row pixel offset, invalidate stale index pages, and reject obsolete results.
- `NativeWindowResourcePolicy::ReleaseViewWhenHidden` releases the View tree, draw and hit plans, shaped-text cache, and transient input state while a window is hidden or minimized, then rebuilds from retained application state when shown.
- Win32, AppKit, and GTK4 share the hide/show lifecycle contract. Retaining the View remains the backward-compatible default.
- The crates.io package contains the source, build assets, examples, and documentation required by consumers while excluding video-production material.

Default features remain `window`, `button`, and `label`. The complete component set, image preview, and advanced desktop capabilities remain explicit Cargo feature choices.
