# ZSUI v0.2.0-preview.6

## 中文

本预览版集中统一三平台应用写法与框架内部平台体验边界，并建立真实 AppKit、Win32、X11 和 Wayland 界面证据。

- 同一套 `State / Msg / view / update` 应用代码可进入 Win32、AppKit 与 Linux 原生窗口宿主，应用层不需要平台枚举、条件编译或原始平台句柄。
- 平台组件度量、字体、导航、标签页、弹层、菜单与交互习惯集中到框架维护的 experience/profile 层；共享 View 树保持平台中立。
- Native UI Proof CI 在固定平台 Runner 上运行真实窗口，导出最终平台表面截图、布局、焦点、事件、字体与运行内存报告。
- AppKit 使用真实 `NSApplication`、`NSWindow`、`NSView`、Core Text 与原生菜单/文件服务；Linux 提供真实 Wayland/X11 窗口、菜单、IME 与 AT-SPI 路径。
- Linux 新增不链接 Cairo/Pango 的 `linux-direct-lite` 纯 Rust 渲染配置，并保留默认 Cairo/Pango 配置；两者复用相同应用代码。
- 三平台文字塑形、双向文本、插入点、选择、滚动与平台字体度量继续由共享协议和各平台原生文字栈协同处理。
- Windows GDI 文字遵循系统消息字体与 ClearType 规则，中文安装环境自动使用系统配置的中文 UI 字体，避免固定 Segoe UI Variable 造成的字形发薄、间距异常和裁剪错觉。
- 组件、服务、后端和高级能力继续通过 Cargo feature 按需编译；默认 feature 保持为 `window`、`button` 和 `label`。

本版本仍为预览版。真实中文输入法候选窗、系统辅助技术和更多桌面环境的人工验收仍是后续发布门槛。

## English

This preview consolidates the shared three-platform application model and the framework-owned platform experience boundary while adding real AppKit, Win32, X11, and Wayland evidence.

- The same `State / Msg / view / update` application code enters native Win32, AppKit, and Linux window hosts without application-level platform enums, conditional compilation, or raw platform handles.
- Platform metrics, typography, navigation, tabs, overlays, menus, and interaction conventions are centralized in framework-owned experience/profile layers while the shared View tree remains platform-neutral.
- Native UI Proof CI runs real windows on fixed platform runners and exports final platform-surface screenshots plus layout, focus, event, typography, and process-memory reports.
- AppKit uses real `NSApplication`, `NSWindow`, `NSView`, Core Text, and native menu/file services. Linux provides real Wayland/X11 window, menu, IME, and AT-SPI paths.
- Linux gains an optional pure-Rust `linux-direct-lite` renderer that does not link Cairo/Pango while preserving the default Cairo/Pango profile; both reuse identical application code.
- Shared protocols and native platform text stacks cooperate on shaping, bidirectional text, carets, selection, scrolling, and platform font metrics.
- Windows GDI text follows the configured system message font and ClearType quality, so Chinese installations use their native UI fallback instead of forcing Segoe UI Variable glyph fallback.
- Components, services, backends, and advanced capabilities remain Cargo-feature gated. Default features remain `window`, `button`, and `label`.

This is still a preview release. Manual validation with real Chinese IME candidate windows, system assistive technologies, and additional desktop environments remains a later release gate.
