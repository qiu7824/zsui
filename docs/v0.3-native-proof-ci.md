# ZSUI 0.3.0: Native Proof CI

## 版本定义

ZSUI 0.3.0 的核心里程碑是建立可重复、可审查、可阻止回归的三平台原生运行证据，
而不是继续以组件数量作为主要完成指标。

每个影响界面的提交必须在目标操作系统上启动真实原生应用，执行固定交互场景，
从最终平台视图导出截图与结构化报告，并通过语义、布局和视觉回归检查。

第一优先级是 GitHub 托管 `macos-15` Runner 上的 AppKit 自动验收。该 Runner
当前是标准 M1 ARM64 虚拟机；公开仓库使用标准 GitHub 托管 Runner 不需要租用
长期云 Mac。`macos-latest` 不得用于视觉基准任务。

## 成功路径

```text
提交或拉取请求
  -> GitHub 创建全新的 macOS 15 ARM64 虚拟机
  -> 编译 ZSUI AppKit 后端和验收应用
  -> 启动真实 NSApplication、NSWindow 和 ZSUI NSView
  -> 执行固定尺寸、主题、数据、焦点和输入场景
  -> 从 NSView 最终显示内容导出 PNG
  -> 写出布局、焦点、事件、文本输入和生命周期 JSON
  -> 严格比较语义与几何
  -> 容差比较关键区域和完整窗口像素
  -> 上传实际图、基准图、差异图和摘要
  -> 通过或阻止合并
```

现有 `cargo test`、后端 `cargo check`、示例编译和 Cargo feature 矩阵继续保留。
Native Proof CI 是新增的目标运行门禁，不能替代现有静态和单元测试。

## 证据等级

ZSUI 0.3.0 继续区分三种状态：

1. `code-level`：代码、单元测试和目标编译通过。
2. `target-runtime`：真实平台进程启动、交互、绘制、截图并正常退出。
3. `system-complete`：自动证据与发布前真实设备人工验收均通过。

`cargo check --features macos-appkit` 只能证明第一层。共享 `DrawPlan` 的离屏 PNG
也只能证明平台中立绘制计划，不能作为 AppKit 目标截图。

## AppKit P0 验收门槛

### 真实应用运行

验收进程必须真实完成：

- 创建并运行 `NSApplication`；
- 创建 `NSWindow` 和 ZSUI 自有 `NSView`；
- 接收真实 content bounds、backing scale factor 和外观；
- 完成首次布局、绘制和事件循环轮转；
- 执行确定性的指针、键盘、焦点、文本和窗口尺寸场景；
- 在限定时间内关闭窗口并正常退出；
- `panic`、未处理命令、截图失败、进程超时和非零原生资源计数均使任务失败。

单次进程退出不能证明不存在所有内存泄漏。Native Proof 必须记录 ZSUI 所有权范围内
的窗口、视图、计时器、跟踪区域、菜单目标和图片表示生命周期计数；关闭后的非零计数
是回归，系统级长期泄漏仍由发布前 Instruments 或等价人工探针补充。

### 最终视图截图

AppKit 截图必须在主线程从最终 `NSView` 生成：

```text
NSView.layoutSubtreeIfNeeded
  -> NSView.displayIfNeeded
  -> bitmapImageRepForCachingDisplayInRect
  -> cacheDisplayInRect:toBitmapImageRep:
  -> NSBitmapImageRep PNG representation
```

截图实现属于 `macos_appkit_renderer`/AppKit host 边界，公开 API 不暴露 Objective-C
对象。捕获前必须确认窗口已附着、布局已稳定、待处理绘制已提交；不得使用固定长时间
`sleep` 冒充首帧同步。

该路径用于验证：

- AppKit 坐标方向和裁剪；
- backing scale 与像素尺寸；
- Core Text 字形和系统字体；
- `NSView` 最终颜色、背景与子视图合成；
- 平台渲染器而非共享绘制计划的实际输出。

### 固定场景

AppKit 核心套件至少生成：

```text
target/native-proof/macos-15-arm64/
  manifest.json
  gallery-inputs-light.png
  gallery-inputs-light.json
  gallery-inputs-dark.png
  gallery-inputs-dark.json
  gallery-collections.png
  gallery-collections.json
  gallery-navigation.png
  gallery-navigation.json
  gallery-feedback.png
  gallery-feedback.json
  gallery-overlays.png
  gallery-overlays.json
  notepad-empty.png
  notepad-empty.json
  notepad-text-selection.png
  notepad-text-selection.json
  notepad-bidi.png
  notepad-bidi.json
  resized-small.png
  resized-small.json
  resized-large.png
  resized-large.json
  shutdown.json
```

每个场景固定：

- 窗口逻辑尺寸和预期像素尺寸；
- 页面、主题和系统外观；
- 测试数据、locale 和文本方向；
- 焦点控件、选择范围和鼠标位置；
- 输入消息序列；
- 动画时间、光标闪烁和计时器时钟；
- 弹层打开状态和锚点；
- 调整尺寸后的最终 content bounds。

系统语言、外观、时钟和动画由 proof 配置显式控制。日期、随机数、光标闪烁和
进行中的动画不得进入稳定基准；能冻结的内容不得依赖遮罩。

### 结构化报告

每个场景的 JSON 使用版本化 schema，至少包含以下字段。示例数值只说明 schema；
Scale Factor 和像素尺寸必须由首次 Runner 探针实测后写入基准：

```json
{
  "schema_version": 1,
  "scenario": "gallery-inputs-light",
  "platform": "macos",
  "backend": "appkit",
  "os_family": "macos-15",
  "architecture": "arm64",
  "runner": {
    "image_os": "macos15",
    "image_version": "recorded-for-diagnostics"
  },
  "scale_factor": 2.0,
  "window": {
    "logical_width": 960,
    "logical_height": 640,
    "pixel_width": 1920,
    "pixel_height": 1280
  },
  "focused_widget": "search-box",
  "widgets": [
    {
      "id": "search-box",
      "role": "textbox",
      "x": 224,
      "y": 82,
      "width": 480,
      "height": 28,
      "enabled": true,
      "focused": true
    }
  ],
  "messages": ["SearchFocused", "SearchChanged"],
  "unhandled_commands": [],
  "errors": []
}
```

`macos-15` 固定操作系统主版本和 ARM64 架构，但 GitHub 仍会更新系统补丁、Xcode
和 Runner 镜像。`image_version`、系统 build 和字体信息必须记录为诊断元数据，不能
假装固定 label 会永久固定每个抗锯齿像素。

首次 Runner 探针必须先记录实际 backing scale；确认稳定后再将预期值写入
`macos-15-arm64` 基准。不得在没有目标机证据时预设 Scale Factor。

## 三层回归判断

### 1. 语义与布局

以下内容严格比较：

- 场景集合和 schema 版本；
- backend、操作系统主版本和架构；
- 窗口逻辑尺寸、像素尺寸和 Scale Factor；
- Widget ID、角色、顺序、启用状态和焦点状态；
- 控件几何，允许误差为 `0 px`；
- 文本选择、IME 协议状态、消息序列和命令结果；
- 错误、未处理事件和生命周期计数必须为空或为零。

Runner 镜像补丁版本只参与诊断，不作为语义相等条件。

### 2. 关键控件区域

关键区域由结构化 Widget bounds 派生，不在比较器中复制一套手写坐标。比较器对
RGBA 像素做统一色彩空间解码，忽略 PNG 元数据，并输出：

- 超出通道容差的像素数；
- 区域差异率；
- 最大和平均颜色差；
- 高亮差异图。

文字边缘可以使用小范围颜色容差，但控件边界、焦点环、裁剪边缘和选区几何不允许
位移。

### 3. 完整窗口

完整窗口采用差异率门槛并生成热力图。阈值存储在基准 manifest 的每个场景中，
不硬编码在框架或工作流中。

阈值由同一提交在至少三台全新 Runner VM 上重复运行后反推：

1. 记录稳定噪声上限；
2. 增加最小安全余量；
3. 关键区域差异率不得高于 `0.5%`；
4. 完整窗口差异率不得高于 `1%`；
5. 若稳定噪声已超过上限，先消除不确定性，不能放宽门槛掩盖问题。

上述百分比是允许的最大上限，不是未经测量即可采用的默认值。

## Native Proof 工具边界

比较和报告逻辑放在独立工具中，不进入 ZSUI 运行时依赖图：

```text
tools/native-proof/
  Cargo.toml
  Cargo.lock
  src/
    main.rs
    compare.rs
    report.rs
    image_diff.rs
```

在根包尚未转换为 Cargo workspace 时使用独立 manifest：

```bash
cargo run --locked \
  --manifest-path tools/native-proof/Cargo.toml \
  -- compare \
  --baseline tests/native-baselines/macos-15-arm64 \
  --actual target/native-proof/macos-15-arm64 \
  --diff target/native-proof-diff/macos-15-arm64
```

工具负责：

- 校验必需截图和 JSON 是否齐全；
- 验证 schema、场景清单和 PNG 尺寸；
- 严格比较语义、焦点、消息和控件边界；
- 按场景比较关键区域和完整窗口；
- 生成 actual/baseline/diff 三联图和 Markdown 摘要；
- 任何缺失、结构回归或超阈值差异返回非零退出码。

生成场景属于示例和平台 host；比较器只消费文件，不启动应用，也不持有 AppKit
对象。

## 基准管理

基准目录：

```text
tests/native-baselines/
  macos-15-arm64/
    manifest.json
    gallery-inputs-light.png
    gallery-inputs-light.json
    ...
```

CI 永远只读基准，不得自动覆盖。基准变更流程固定为：

```text
代码变更
  -> CI 生成 actual
  -> 比较失败并上传 diff
  -> 人工确认视觉变化
  -> 单独提交基准变更
  -> 新旧证据在拉取请求中共同审查
```

基准提交必须与行为修改分离，提交标题使用明确的目标和系统版本，例如：

```text
test: 更新 macOS 15 ARM64 AppKit 原生视觉基准
```

## 目标工作流

`.github/workflows/native-proof.yml` 首先强制运行 AppKit Gallery 与 Notepad、校验
最终 `NSView` PNG 和捕获元数据并上传证据。完成场景集、比较器和首批基准后，工作流
扩展为以下完整回归门禁：

```yaml
name: Native UI Proof

on:
  push:
    branches: [main]
    paths:
      - "src/**"
      - "examples/**"
      - "tests/native-baselines/**"
      - "tools/native-proof/**"
      - "Cargo.toml"
      - "Cargo.lock"
      - ".github/workflows/native-proof.yml"
  pull_request:
    paths:
      - "src/**"
      - "examples/**"
      - "tests/native-baselines/**"
      - "tools/native-proof/**"
      - "Cargo.toml"
      - "Cargo.lock"
      - ".github/workflows/native-proof.yml"
  workflow_dispatch:

permissions:
  contents: read

concurrency:
  group: native-proof-${{ github.ref }}
  cancel-in-progress: true

jobs:
  macos-appkit-proof:
    name: AppKit Runtime and Visual Proof
    runs-on: macos-15
    timeout-minutes: 30
    env:
      LANG: en_US.UTF-8
      LC_ALL: en_US.UTF-8

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Install Rust
        uses: dtolnay/rust-toolchain@stable

      - name: Restore Rust cache
        uses: Swatinem/rust-cache@v2

      - name: Verify fixed runner identity
        run: |
          test "$(sw_vers -productVersion | cut -d. -f1)" = "15"
          test "$(uname -m)" = "arm64"
          printf 'ImageOS=%s\nImageVersion=%s\n' \
            "$ImageOS" "$ImageVersion"

      - name: Build AppKit proof applications
        run: |
          cargo build --release --locked \
            --example component_gallery \
            --no-default-features \
            --features component-gallery-demo
          cargo build --release --locked \
            --example zsui_notepad \
            --no-default-features \
            --features notepad-demo

      - name: Run AppKit native proof
        run: |
          mkdir -p target/native-proof/macos-15-arm64
          ./target/release/examples/component_gallery \
            --native-proof \
            --suite appkit-core \
            --output target/native-proof/macos-15-arm64
          ./target/release/examples/zsui_notepad \
            --native-proof \
            --suite appkit-core \
            --output target/native-proof/macos-15-arm64

      - name: Compare native proof
        run: |
          cargo run --locked \
            --manifest-path tools/native-proof/Cargo.toml \
            -- compare \
            --baseline tests/native-baselines/macos-15-arm64 \
            --actual target/native-proof/macos-15-arm64 \
            --diff target/native-proof-diff/macos-15-arm64

      - name: Upload proof and differences
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: zsui-appkit-native-proof
          path: |
            target/native-proof/macos-15-arm64
            target/native-proof-diff/macos-15-arm64
          if-no-files-found: warn
          retention-days: 30
```

工作流首次落地时先通过 `workflow_dispatch` 运行 Runner 探针和基准候选。只有基准
经过独立提交审查后，才将 `AppKit Runtime and Visual Proof` 配置为分支保护必需检查。

## 三平台交付顺序

### 0.3.0-alpha.1：AppKit P0

- 固定 `macos-15` ARM64 Runner；
- 实现最终 `NSView` PNG 捕获；
- Gallery 全场景和 Notepad 文本场景运行；
- 结构化报告、比较器、差异图和只读基准；
- AppKit proof 成为拉取请求必需检查。

### 0.3.0-beta.1：Win32 统一证据

- 将现有 Windows smoke 迁移到同一 proof schema；
- 在固定 Windows Runner 上生成 Gallery/Notepad 对应证据；
- 保留 GDI 缓冲无闪烁和 GUI 子系统检查；
- Win32 proof 成为必需检查。

### 0.3.0-rc.1：GTK4 统一证据

- 在固定 Ubuntu Runner 和明确记录的 X11/Xvfb 或 Wayland 会话中启动真实
  `GtkApplication`；
- 从 GTK4 最终绘制 surface 导出截图；
- 复用场景 schema、比较器、基准管理和差异上传；
- GTK4 proof 成为必需检查。

### 0.3.0 正式版

三平台 proof 均为绿色且分支保护生效。`macos-26` 可作为发布工作流的独立矩阵，
但在拥有单独、经审查的基准前不得复用 `macos-15` 像素基准或阻塞每次提交。

## IME 与辅助功能边界

GitHub Runner 自动验证：

- `NSTextInputClient` 方法路由；
- `setMarkedText`、`insertText`、取消、删除和提交；
- 预编辑文本、选区、光标与候选窗口锚点数据；
- AppKit 可访问角色、名称、状态和焦点树的结构输出。

Runner 不能替代：

- 真实中文输入法候选窗的选择与定位体验；
- VoiceOver 的完整导航和朗读体验；
- 触控板、真实显示器色彩和多屏 Retina 行为。

0.3.0 发布前必须在真实 Mac 上补充中文输入法、VoiceOver、缩放和窗口生命周期
人工验收；自动协议覆盖不得描述为真实候选窗或辅助技术完成证据。

## 0.3.0 发布门槛

- AppKit Gallery 在 `macos-15` Runner 上真实启动并输出最终视图截图；
- AppKit Notepad 自动完成首帧、输入、选择、双向文本和缩放探针；
- 浅色、深色、输入、集合、导航、反馈、弹层和尺寸场景齐全；
- 截图来自平台最终视图，不来自共享 `DrawPlan`；
- 焦点、布局、消息、文本输入和生命周期报告齐全；
- 语义和几何回归严格失败，视觉回归按校准阈值失败；
- 失败任务始终上传 actual、baseline、diff 和 Markdown 摘要；
- 基准更新只能通过独立人工审查提交；
- 现有编译、测试、feature 矩阵和平台边界检查继续通过；
- Win32、AppKit 和 GTK4 使用同一 proof schema 并成为必需检查；
- 发布前真实 Mac IME 与辅助功能人工验收完成。

## 明确不做

- 不为 0.3.0 P0 租用长期云 Mac；
- 不使用 `macos-latest` 建立基准；
- 不把共享 DrawPlan PNG 描述为平台截图；
- 不要求所有抗锯齿像素完全相等；
- 不自动覆盖或接受基准；
- 不用放宽阈值掩盖动画、时钟、字体或布局不确定性；
- 不把协议级 IME 测试描述为真实输入法候选窗验收；
- 不把 Native Proof 比较器加入 ZSUI 运行时依赖。
